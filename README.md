## рҹҢҹ What is Style Lovin Logins?

> **The world's most obsessive collection of login UI designs.**

Just raw **HTML + CSS + JS** вҖ” open a file and it works.

## вҡЎ Quick Start

```bash
# 1пёҸвғЈ  Clone the repo
git clone https://github.com/style-lovin/style-lovin-logins.git
cd style-lovin-logins

# 2пёҸвғЈ  Install dev tools (optional вҖ” only for scripts & linting)
npm install

# 3пёҸвғЈ  Open any design directly вҖ” no server needed!
open login-designs/glassmorphism-dark/index.html

# 4пёҸвғЈ  Or launch the interactive preview server
npm run preview
# вҶ’ http://localhost:3000  (pick any design from dropdown)

# 5пёҸвғЈ  Scaffold a brand new design
node scripts/new-design.js my-sick-design
```

## рҹӣ пёҸ Available Commands

| Command | What it does |
|---|---|
| `npm run dev` | Live-server with hot reload |
| `npm run preview` | Interactive design selector at localhost:3000 |
| `npm run validate` | Lint all designs for missing files & broken rules |
| `npm run count` | Count total designs + show progress to 1000 |
| `npm run generate-index` | Build `dist/index.html` вҖ” the full gallery |
| `npm run lint` | ESLint on all JS files |
| `npm run lint:css` | Stylelint on all CSS files |
| `npm run format` | Prettier format everything |
| `npm test` | Run all automated tests |
| `node scripts/new-design.js <name>` | Scaffold a new design instantly |

## вһ• Adding Your Own Design

Creating a design is **dead simple**:

```bash
# Instant scaffold вҖ” creates index.html, style.css, script.js
node scripts/new-design.js your-design-name

# Then open and go wild:
open login-designs/your-design-name/index.html
```

Your folder should look like this:

```
login-designs/
в””в”Җв”Җ your-design-name/
    в”ңв”Җв”Җ index.html   вҶҗ The page
    в”ңв”Җв”Җ style.css    вҶҗ Your creative chaos
    в”ңв”Җв”Җ script.js    вҶҗ Animations, interactions (optional)
    в””в”Җв”Җ preview.png  вҶҗ Screenshot 1280x720 (helps in gallery)
```

> **Rule #1:** It must work by just opening `index.html` in a browser. No build steps, no npm, no excuses.

See [Creating a Design](docs/guides/creating-a-design.md) for the full guide.
